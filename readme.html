Sales ERP

Laravel 12 · React 18 · Inertia.js · Tailwind CSS · Multi-tenant SaaS

Laravel 12 React 18 Inertia.js 2 Multi-tenant Version 1.0.0

Overview

Sales ERP is a comprehensive, multi-tenant enterprise resource planning system designed for small to medium businesses. It covers the full order-to-cash lifecycle including Sales, CRM, Purchasing, Inventory, Accounting, HR & Payroll, Fixed Assets, POS, and advanced Industrial Engineering analytics — all in a single, modern web application.

The frontend is built with React 18 and Inertia.js, providing a SPA-like experience without a separate API. The backend is Laravel 12 with full multi-tenancy: each company's data is isolated. Permissions are controlled through a flexible SuperAdmin → Admin → Role hierarchy.

Installation

  1. Upload files

    Extract the ZIP and upload all files to your server. The web server root must point to the public/ sub-directory, not the project root.

  2. Create the database

    Create a new MySQL database (e.g., sales_erp) and note the credentials.

  3. Configure environment

    Copy .env.example to .env and fill in your values: 

    cp .env.example .env

    Key settings to update in .env:

    APP_NAME="Sales ERP"
APP_ENV=production
APP_DEBUG=false
APP_URL=https://your-domain.com

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=sales_erp
DB_USERNAME=db_user
DB_PASSWORD=db_password

  4. Install PHP dependencies

    composer install --optimize-autoloader --no-dev

  5. Generate application key

    php artisan key:generate

  6. Run migrations & seeders

    php artisan migrate --seed
    Default Login: Email: admin@example.com  |  Password: Admin@1234
    Change these immediately after first login.

  7. Build frontend assets

    npm install
npm run build

  8. Set directory permissions

    chmod -R 775 storage bootstrap/cache
php artisan storage:link

  9. Optimize for production

    php artisan config:cache
php artisan route:cache
php artisan view:cache

Web Server Configuration

Apache

<VirtualHost *:80>
    ServerName your-domain.com
    DocumentRoot /var/www/sales_erp/public

    <Directory /var/www/sales_erp/public>
        AllowOverride All
        Require all granted
    </Directory>
</VirtualHost>

Enable mod_rewrite: a2enmod rewrite && systemctl reload apache2

Nginx

server {
    listen 80;
    server_name your-domain.com;
    root /var/www/sales_erp/public;
    index index.php;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location ~ \.php$ {
        fastcgi_pass unix:/var/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}

Queue Worker

Email notifications and background tasks run through Laravel Queues. Start a worker: 

php artisan queue:work --queue=default --tries=3 --sleep=3

For production, manage it with Supervisor: 

[program:sales_erp_queue]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/sales_erp/artisan queue:work --tries=3
autostart=true
autorestart=true
numprocs=2
user=www-data
redirect_stderr=true
stdout_logfile=/var/log/sales_erp_queue.log

Mail Setup

Configure your SMTP credentials in .env:

MAIL_MAILER=smtp
MAIL_HOST=smtp.yourprovider.com
MAIL_PORT=587
MAIL_USERNAME=your@email.com
MAIL_PASSWORD=yourpassword
MAIL_ENCRYPTION=tls
MAIL_FROM_ADDRESS="noreply@your-domain.com"
MAIL_FROM_NAME="${APP_NAME}"
Popular options: Mailgun, Amazon SES, SendGrid, Mailtrap (for testing).

First Login

After running php artisan migrate --seed, log in at https://your-domain.com/login:

Field Value
Email admin@example.com
Password Admin@1234
Important: Change the default password immediately after first login via Profile → Change Password.

From the SuperAdmin panel you can:

  • Update company name and settings
  • Create tenant Admin users and assign plans
  • Manage subscription plans and features

Multi-tenancy

Sales ERP uses a shared database, shared schema multi-tenancy model. Each company (tenant) is identified by a company_id foreign key on all data tables. All queries are automatically scoped to the authenticated user's company.

  • SuperAdmin — system owner; manages all companies and plans.
  • Admin — tenant owner; manages their company's users and data.
  • Sub-users — created by Admin with specific role-based permissions.

The Plans & Subscriptions module lets you gate features per plan tier.

Roles & Permissions

Permissions are defined per module and feature. The delegation chain is: 

SuperAdmin  →  defines base permissions per tenant
   └── Admin    →  creates roles and can only grant own permissions
         └── Sub-users  →  assigned roles, access is strictly scoped

Manage roles at: Settings → Roles & Permissions

A tenant Admin cannot create or modify other Admin or SuperAdmin accounts.

Upgrading

  1. Back up your database and .env file.
  2. Replace all project files (keep your .env).
  3. Run: composer install --optimize-autoloader --no-dev
  4. Run: php artisan migrate
  5. Run: npm install && npm run build
  6. Run: php artisan config:cache && php artisan route:cache && php artisan view:cache

Troubleshooting

Problem Solution
White/blank page after install Check APP_DEBUG=true temporarily, check storage/logs/laravel.log. Usually a missing APP_KEY or wrong DB credentials.
500 error on API calls Ensure storage/ and bootstrap/cache/ are writable (chmod -R 775).
Assets not loading (404 on /build/...) Run npm run build — the public/build/ folder must exist.
Emails not sending Verify MAIL_* settings and that a queue worker is running for non-sync mailers.
Migrations fail Make sure DB user has CREATE/ALTER privileges. Check DB_DATABASE exists.
Session / login loop Set SESSION_DOMAIN to your domain (without http), ensure cookies are not blocked.

Always check storage/logs/laravel.log for detailed error messages.
